# Nacos的docker启动参数

详细解释与填写方法
1. NACOS_AUTH_TOKEN （必须填写）
这是 Nacos 用来生成 JWT 的密钥。

要求：长度大于 32 字符，并且要经过 Base64 编码。

示例：
假设你想用这个字符串作为密钥：

复制
编辑
nacosSuperSecretToken1234567890!
你需要先用 Base64 编码：

bash
复制
编辑
echo -n nacosSuperSecretToken1234567890! | base64
输出：

text
复制
编辑
bmFjb3NTdXBlclNlY3JldFRva2VuMTIzNDU2Nzg5MCE=
你就应该写成：

bash
复制
编辑
-e NACOS_AUTH_TOKEN=bmFjb3NTdXBlclNlY3JldFRva2VuMTIzNDU2Nzg5MCE=
2. NACOS_AUTH_IDENTITY_KEY （必须填写）
这是用来做 Nacos 集群间内部通信身份校验的 key。

单节点可以随便设置，建议用：

text
复制
编辑
nacos-node-key
3. NACOS_AUTH_IDENTITY_VALUE （必须填写）
这是用来做 Nacos 集群间内部通信身份校验的 value。

单节点可以随便设置，建议用：

text
复制
编辑
nacos-node-value
完整示例指令
bash
复制
编辑
docker run --name nacos-standalone-derby \
    -e MODE=standalone \
    -e NACOS_AUTH_TOKEN=bmFjb3NTdXBlclNlY3JldFRva2VuMTIzNDU2Nzg5MCE= \
    -e NACOS_AUTH_IDENTITY_KEY=nacos-node-key \
    -e NACOS_AUTH_IDENTITY_VALUE=nacos-node-value \
    -p 8080:8080 \
    -p 8848:8848 \
    -p 9848:9848 \
    -d nacos/nacos-server:latest

重点总结
参数	是否必须	如何填写	示例
NACOS_AUTH_TOKEN	必须	大于32位，Base64编码	bmFjb3...
NACOS_AUTH_IDENTITY_KEY	必须	任意字符串	nacos-node-key
NACOS_AUTH_IDENTITY_VALUE	必须	任意字符串	nacos-node-value
